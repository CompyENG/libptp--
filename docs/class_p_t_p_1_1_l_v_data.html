uint8_t * PTP::LVData::get_rgb ( int *  out_size,
int *  out_width,
int *  out_height,
bool  skip = false 
)

Get live view data in RGB format.

This function is quite a doozy, and likely to be the most CPU-intense task an LVData can perform. This method takes the live view data (in YUV format) from the payload and converts it to RGB so that it can actually be used. The skip parameter will depend on which camera is used. Size, width, and height are calculated from properties of the live view data, to hide the underlying structure.

Warning
This function malloc()s space for the resulting data. Be sure to free() it!
Parameters
[out]out_sizeThe size of the resulting RGB data
[out]out_widthThe width of the resulting RGB image
[out]out_heightThe height of the resulting RGB image
[in]skipIf true, skips two pixels of every four (required on some cameras)
Returns
The address of the first byte of the resulting RGB image
See Also
http://chdk.wikia.com/wiki/Frame_buffers#Viewport, http://trac.assembla.com/chdk/browser/trunk/tools/yuvconvert.c

void PTP::LVData::read ( uint8_t *  payload,
int  payload_size 
)

Read data from payload into the live view data structures.

Parases payload for the necessary parts of the payload and places them in our internal structures. Also stores a copy of the complete payload for later use in retrieving data. This way, we only spend CPU time on the data retrieval we NEED to make.

Parameters
[in]payloadThe address of the first byte of a PTP payload
[in]payload_sizeThe number of bytes in the payload
Exceptions
LVDATA_NOT_ENOUGH_DATAIf payload_size given cannot possibly be large enough to actually contain live view data.

Read live view data directly from a PTPContainer.

This function exists so that we can hide the actual payload data from calling functions, and just pass PTPContainer s around. In a perfect data-hiding world, PTPContainer and LVData might be friend classes, so that methods to access the payload could be kept protected.
