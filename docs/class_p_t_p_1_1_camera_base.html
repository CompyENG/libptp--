void PTP::CameraBase::ptp_transaction ( PTPContainer *  cmd,
PTPContainer *  data,
bool  receiving,
PTPContainer *  out_resp,
PTPContainer *  out_data,
int  timeout = 0 
)

Perform a complete write, and optionally read, PTP transaction.

At minimum, it is required that cmd is not NULL. All other containers are checked for NULL values before reading/writing. Note that this function will also modify cmd and data to place a generated transaction ID in them, required by the PTP protocol.

Although not enforced by this function, cmd should be a PTPContainer containing a command, and data (if given) should be a PTPContainer containing data.

Note that even if receiving is false, PTP requires that we receive a response. If provided, out_resp will be populated with the command response, even if receiving is false.

Warning
CameraBase::_bulk_read and CameraBase::_bulk_write are called multiple times during the execution of this function, and timeout is passed to each of them individually. Therefore, this function could take much more than timeout seconds to return.
Parameters
[in]cmdA PTPContainer containing the command to send to the camera.
[in]data(optional) A PTPContainer containing the data to be sent with the command.
[in]receivingWhether or not to receive data in addition to a response from the camera.
[out]out_resp(optional) A PTPContainer where the camera's response will be placed.
[out]out_data(optional) A PTPContainer where the camera's data response will be placed.
[in]timeoutThe maximum number of seconds each CameraBase::_bulk_read or CameraBase::_bulk_write should attempt to communicate for.
See Also
CameraBase::send_ptp_message, CameraBase::recv_ptp_message

void PTP::CameraBase::recv_ptp_message ( PTPContainer *  out,
int  timeout = 0 
)

Recives a PTPContainer from the camera and returns it.

This function works by first reading in a buffer of 512 bytes from the camera to determine the length of the PTP message it will receive. If necessary, it then makes another CameraBase::_bulk_read call to read in the rest of the data. Finally, PTPContainer::unpack is called to place the data in out.

Warning
timeout is passed to each call to CameraBase::_bulk_read. Therefore, this function could take up to 2 * timeout seconds to return.
Parameters
[out]outA pointer to a PTPContainer that will store the read PTP message.
[in]timeoutThe maximum number of seconds to wait to read each time.
See Also
CameraBase::_bulk_read, CameraBase::send_ptp_message
